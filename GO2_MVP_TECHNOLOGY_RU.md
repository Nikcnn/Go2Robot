# Go2 & D1 Inspection System: техническое состояние проекта

Этот файл описывает текущее состояние проекта. Система объединяет мобильную платформу Unitree Go2 и манипулятор Unitree D1.

## 1. Что входит в проект

В репозитории три ключевых направления:

- `src/`: Python-приложение (FastAPI, панель оператора, исполнение маршрутов, интеграция с D1 через UNIX-сокет).
- `cpp/d1_bridge/`: C++17 демон-бридж для управления и мониторинга манипулятора D1.
- `ros_ws/`: ROS 2 Foxy workspace для навигации через Nav2.

## 2. Платформа

- OS: Ubuntu 20.04 (целевая для ROS и real-hardware).
- Python: 3.8.
- C++: C++17 (для d1_bridge).
- ROS: ROS 2 Foxy.
- RMW: `rmw_cyclonedds_cpp`.

## 3. Структура репозитория

```text
src/                  Python application layer
src/integrations/     Клиенты для внешних сервисов (D1 Bridge)
src/services/         Фоновые сервисы (D1 monitoring/control)
cpp/d1_bridge/        C++ демон для манипулятора D1
ros_ws/               ROS 2 Foxy workspace
config/routes/        Scripted routes для Go2
shared_missions/      Coordinate missions для ROS
runs/                 Runtime artifacts
tests/                Hardware-free tests
```

## 4. Интеграция с Unitree D1 Arm

Интеграция реализована через отдельный сервис-бридж на C++:
- **Протокол**: JSON-сообщения через UNIX domain socket (`/run/d1_bridge.sock`).
- **Состояние**: Реализован полный цикл мониторинга и управления. Движение манипулятора защищено многоуровневой системой безопасности (Gating).
- **Интерфейс**: В панели оператора доступна вкладка "D1 Arm" для управления суставами, калибровки и мониторинга в реальном времени.

## 5. Python Application Layer

Приложение на FastAPI обеспечивает:
- Центральное управление роботом Go2.
- Управление и мониторинг манипулятора D1 через `D1Service`.
- Потоковое видео с камер.
- Хранение отчетов и логов миссий.

## 6. Mock Mode

Проект поддерживает полный Mock-режим:
- **Go2**: Синтетическая телеметрия и позиция.
- **D1 Arm**: Полная имитация движений и обратной связи от d1_bridge.
- **Sensors**: Тестовые кадры камеры.

Это позволяет вести разработку и тестирование логики без доступа к физическому оборудованию.

## 7. Control Semantics (Безопасность)

Приоритет управления:
```text
ESTOP > MANUAL > AUTO
```
- ESTOP мгновенно блокирует все движения Go2 и D1.
- Манипулятор D1 требует явного включения (Enable Motion) через интерфейс или API.
- Перехват управления (Manual Takeover) ставит активную миссию на паузу.

## 8. ROS 2 Foxy Layer

ROS-слой отвечает за высокоуровневую навигацию:
- Построение карт (SLAM).
- Локализация (AMCL).
- Планирование пути (Nav2).
- Связь с Go2 через `go2_bridge`.

## 9. Tests

Hardware-free tests:
```bash
pytest
```
Тесты покрывают API, логику приоритетов, управление D1 и навигационные сценарии в Mock-режиме.

## 10. Краткий итог

Проект развился в полноценную гибридную систему "мобильный робот + манипулятор". Архитектура обеспечивает безопасную и эффективную эксплуатацию Go2 и D1 в рамках единого интерфейса управления.
